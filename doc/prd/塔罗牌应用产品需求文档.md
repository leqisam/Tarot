## 1. 文档概述

### 1.1 文档目的

本文档旨在详细描述塔罗牌抽卡预测应用的产品需求，为开发团队、设计团队提供清晰的功能规格和实现指导。

### 1.2 产品背景

塔罗牌是一种古老的占卜工具，通过抽取卡牌来探索问题的答案和人生的指引。随着人们对心理健康、自我探索的需求增加，塔罗牌逐渐从神秘学走向大众娱乐和心理辅助领域。本产品旨在通过数字化方式，让用户随时随地获得塔罗占卜体验。

### 1.3 术语说明

| 术语 | 说明 |
|-----|------|
| 大阿尔卡纳 | 塔罗牌中的22张主牌，代表重大人生课题 |
| 小阿尔卡纳 | 塔罗牌中的56张副牌，分为权杖、圣杯、宝剑、星币四组 |
| 正位 | 卡牌正常方向，代表正面含义 |
| 逆位 | 卡牌倒置方向，代表反面或阻碍含义 |
| 牌阵 | 多张卡牌按特定位置摆放的布局方式 |
| 凯尔特十字 | 经典的10张牌阵，用于深度分析问题 |

## 2. 产品定位与价值

### 2.1 产品定位

**产品名称**：星语塔罗（暂定）

**定位描述**：一款专注于提供沉浸式塔罗占卜体验的应用，结合传统塔罗智慧与现代交互设计，帮助用户在困惑时获得心灵指引。

**slogan**：探索命运的指引，聆听内心的声音

### 2.2 核心价值主张

1. **便捷性**：随时随地进行占卜，无需预约占卜师
2. **专业性**：基于经典塔罗体系，提供准确的牌意解读
3. **仪式感**：精心设计的抽卡动画，营造神秘氛围
4. **个性化**：针对具体问题给出定制化建议

### 2.3 产品目标

- **短期目标**：打造流畅的单张牌占卜体验，验证用户需求
- **中期目标**：建立完整的牌阵系统和牌意知识库
- **长期目标**：成为用户信赖的心灵指引助手，建立塔罗爱好者社区

## 4. 核心用户场景

### 场景1：情感困惑寻求指引

**场景描述**：小美和男友吵架后，心情低落，想知道这段关系是否值得继续。

**用户旅程**：

```
打开应用 → 被首页氛围打动 → 输入感情问题 
    ↓
选择关系牌阵 → 经历抽卡仪式 → 查看牌面结果
    ↓
阅读详细解读 → 获得情感共鸣 → 收藏结果
```

**关键触点**：
- 首页文案引起情感共鸣
- 问题输入有感情类模板可选
- 抽卡过程有仪式感
- 解读内容温暖、有建设性

**成功标准**：用户完成占卜后情绪得到安抚，对关系有新的认识

---

### 场景2：职业选择决策辅助

**场景描述**：小王收到两个offer，不知道如何选择，希望获得一些启示。

**用户旅程**：

```
进入占卜 → 选择凯尔特十字牌阵 → 输入职业选择问题
    ↓
依次抽取10张牌 → 查看每张牌的位置含义
    ↓
综合解读分析 → 对比两个选择的利弊 → 做出决策
```

**关键触点**：
- 牌阵选择有清晰的场景说明
- 每张牌的位置含义明确
- 综合解读有逻辑性
- 给出具体的行动建议

**成功标准**：用户通过占卜理清了思路，做出了选择

---

### 场景3：每日运势查看

**场景描述**：小张每天早起习惯查看今日运势，为一天做好准备。

**成功标准**：用户养成每日查看的习惯

### 5.2 核心占卜模块

#### 5.2.1 问题输入功能

**功能描述**：用户输入想要占卜的问题

**需求规格**：

| 需求项 | 规格说明 |
|-------|---------|
| 输入方式 | 文本输入框，支持多行文本 |
| 字数限制 | 最少5字，最多200字 |
| 问题模板 | 提供热门问题标签（感情、事业、财运等） |
| 输入提示 | 占位符提示"请输入你想问的问题..." |
| 空值校验 | 未输入时提示"请输入问题后再开始占卜" |
| 敏感词过滤 | 过滤不当内容 |

**交互流程**：
1. 用户进入占卜页面，显示问题输入框
2. 用户可直接输入问题，或点击标签快速填充
3. 点击"下一步"进入牌阵选择
4. 输入不合规时显示错误提示

#### 5.2.2 牌阵选择功能

**功能描述**：用户选择适合的牌阵进行占卜

**牌阵列表**：

| 牌阵名称 | 卡牌数量 | 适用场景 | 难度 |
|---------|---------|---------|------|
| 单张牌 | 1张 | 每日运势、简单是非问题 | 简单 |
| 三张牌-时间之流 | 3张 | 过去-现在-未来 | 简单 |
| 三张牌-问题分析 | 3张 | 问题-障碍-建议 | 中等 |
| 关系牌阵 | 5张 | 感情关系分析 | 中等 |
| 职业牌阵 | 7张 | 事业发展分析 | 中等 |
| 凯尔特十字 | 10张 | 复杂问题全面分析 | 困难 |

**需求规格**：

| 需求项 | 规格说明 |
|-------|---------|
| 展示方式 | 卡片式网格布局 |
| 卡片内容 | 牌阵名称、卡牌数量、适用场景描述 |
| 选中状态 | 选中卡片高亮，显示边框 |
| 默认选择 | 默认选中"单张牌" |
| 禁用状态 | 复杂牌阵在未完成基础教程前可标记为"进阶" |

#### 5.2.3 抽卡流程

**功能描述**：完整的抽卡仪式流程，包括洗牌、切牌、抽卡

**流程步骤**：

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  准备   │ → │  洗牌   │ → │  切牌   │ → │  抽卡   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
   显示牌背      洗牌动画       用户切牌      依次翻牌
   营造氛围      3-5秒         点击切分       动画展示
```

**需求规格**：

| 阶段 | 动画描述 | 交互 | 时长 |
|-----|---------|------|------|
| 准备 | 卡牌堆叠在桌面中央 | 自动 | 1秒 |
| 洗牌 | 卡牌随机移动、堆叠 | 自动播放 | 3-5秒 |
| 切牌 | 卡牌分为两堆 | 用户点击切分 | 等待用户 |
| 抽卡 | 卡牌依次翻转飞入牌阵位置 | 自动 | 每张1秒 |

**正逆位机制**：
- 每张牌有50%概率正位，50%概率逆位
- 逆位卡牌在牌阵中显示为倒置
- 解读时根据正逆位显示不同含义

#### 5.2.4 结果展示功能

**功能描述**：展示占卜结果，包括牌面、位置含义、解读

**页面结构**：

```
┌─────────────────────────────────────────┐
│  问题：我和他的关系会走向何方？          │
├─────────────────────────────────────────┤
│                                         │
│           [牌阵可视化区域]               │
│         （按牌阵位置排列卡牌）            │
│                                         │
├─────────────────────────────────────────┤
│  整体解读                                │
│  根据牌阵整体情况给出的综合分析...        │
├─────────────────────────────────────────┤
│  单张牌详情（可展开）                     │
│  ▼ 位置1 - 现状（圣杯二 正位）           │
│    含义：...                             │
│    建议：...                             │
├─────────────────────────────────────────┤
│  [重新占卜]  [保存结果]  [分享]          │
└─────────────────────────────────────────┘
```

**需求规格**：

| 需求项 | 规格说明 |
|-------|---------|
| 牌阵展示 | 按牌阵定义的位置布局显示卡牌 |
| 卡牌显示 | 显示卡牌图片、名称、正逆位状态 |
| 整体解读 | 基于所有牌的综合分析，200-500字 |
| 单张解读 | 每张牌的位置含义+正逆位含义+建议 |
| 展开收起 | 点击卡牌展开详情，再次点击收起 |

## 8. 非功能需求

### 8.1 性能需求

| 指标 | 要求 |
|-----|------|
| 页面首屏加载时间 | < 3秒（3G网络） |
| 抽卡动画帧率 | > 30fps |
| 页面切换响应时间 | < 500ms |
| 接口响应时间 | < 1秒（正常网络） |
| 并发用户数 | 支持1000人同时在线 |

### 8.2 兼容性需求

| 平台 | 版本要求 |
|-----|---------|
| iOS Safari | iOS 12+ |
| Android Chrome | Android 8+ |
| 微信内置浏览器 | 最新版 |
| PC浏览器 | Chrome 80+, Firefox 75+, Safari 13+ |

### 8.3 安全需求

- 用户数据传输使用HTTPS加密
- 敏感操作需要身份验证
- 用户密码加密存储
- 防止SQL注入和XSS攻击
- 接口访问频率限制

### 8.4 可用性需求

- 支持离线浏览已加载内容
- 关键操作有二次确认
- 网络异常有友好提示
- 支持页面返回和状态保持

### 8.5 可维护性需求

- 代码模块化，组件可复用
- 配置与代码分离
- 完善的日志记录
- 支持灰度发布

**用途**：分析问题并获得建议

**布局**：
```
┌─────┐ ┌─────┐ ┌─────┐
│     │ │     │ │     │
│  1  │ │  2  │ │  3  │
│     │ │     │ │     │
└─────┘ └─────┘ └─────┘
 问题    障碍    建议
```

**位置含义**：
- 位置1：问题的本质
- 位置2：面临的障碍
- 位置3：建议的行动

#### 凯尔特十字（Celtic Cross）

**用途**：深度全面分析复杂问题

**布局**：
```
         ┌─────┐
         │  2  │
    ┌────┼─────┼────┐
    │ 1  │  5  │ 3  │
    └────┼─────┼────┘
         │  4  │
         └─────┘

┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
│  6  │ │  7  │ │  8  │ │  9  │ │  10 │
└─────┘ └─────┘ └─────┘ └─────┘
```

**位置含义**：
1. 现状：当前状况的核心
2. 挑战：面临的阻碍或助力
3. 基础：问题的基础/过去
4. 过去：最近过去的影响
5. 目标：期望的结果/未来
6. 未来：近期发展
7. 自我：求问者的态度
8. 环境：外部环境因素
9. 希望/恐惧：内心的期待或担忧
10. 结果：最终可能的结果
